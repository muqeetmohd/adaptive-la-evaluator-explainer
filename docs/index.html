Pipeline

Diagnose First. Explain Second.

Three sequential stages ensure every explanation is grounded in retrieved source material and calibrated to the student's actual level.

1

Diagnostic

Three targeted questions probe known linear algebra misconceptions — commutativity, geometric vs. arithmetic framing, and abstraction level. Rule-based classification routes the student into Tier 1, 2, or 3.

2

Tier-Filtered Retrieval

The student's query is embedded with all-MiniLM-L6-v2 and used to search ChromaDB with a metadata filter: {"tier": t, "topic": q}. Only chunks matching the diagnosed tier and selected topic are retrieved.

3

Grounded Generation

Retrieved chunks are passed to Mistral 7B Instruct via Ollama with a tier-specific system prompt. The model is instructed to use only the provided context and cite its sources in the response.

Tier System

Three Levels of Explanation

Each tier maps to a distinct explanation style, vocabulary level, and source document. The diagnostic determines which tier each student receives.

Tier 1

Geometric Intuition

Plain English, analogies, and physical intuition. No formal notation. Built for students who have not yet encountered transformation language.

"Think of a matrix as a machine — you feed an arrow in and it produces a new arrow according to a fixed recipe."
Tier 2

Formal Beginner

Formal definitions paired with concrete worked examples. Every abstraction is grounded in a specific numerical case.

"For eigenvalue λ₁ = 3: solve (A − 3I)v = 0 to get v₁ = (1, 0). This is an eigenvector."
Tier 3

Algebraically Grounded

Abstract formulation using axioms, subspaces, and structural theorems. Assumes comfort with formal mathematical language.

"The eigenspace E_λ = ker(T − λI) is a nonzero subspace of V. Eigenvectors for distinct eigenvalues are linearly independent."
Results

Sample Outputs

Query: "What is an eigenvector?" — run through the full pipeline for each tier. All three passed the smoke test.

Tier 1 — Geometric Intuition 5 chunks retrieved Source: vector_spaces_notes
An eigenvector is a special kind of vector that, when transformed by a matrix, only changes its size (length) but keeps its direction the same. This is similar to stretching a piece of string when you pull on it — the change in length happens along the same direction as the original string.

Imagine you have a rubber sheet (a vector space), and you stretch it using a transformation represented by a matrix. Most points on this rubber sheet will move to new positions in a complex way. However, there are special directions where the stretching happens only along those directions without changing their orientation. The vectors aligned with these special directions are called eigenvectors.

The amount of stretching along the eigenvector is captured by a number called the eigenvalue. If an eigenvector gets tripled in length, its eigenvalue is 3. If it flips but stays the same length, the eigenvalue is −1.

Sources: vector_spaces_notes
Tier 2 — Formal Beginner 5 chunks retrieved Sources: intro_linear_algebra_notes, systems_row_reduction_notes
An eigenvector is a special vector that, when multiplied by a square matrix A, results in a new vector that is either stretched or flipped but does not change direction. In other words, an eigenvector undergoes only scaling — no rotation — when acted on by the matrix.

Concrete example using A = [[3, 1], [0, 2]]:

The characteristic equation gives det(A − λI) = (3 − λ)(2 − λ) = 0, so eigenvalues are λ₁ = 3 and λ₂ = 2.

For λ₁ = 3: solve (A − 3I)v = 0 → v₁ = (1, 0).
For λ₂ = 2: solve (A − 2I)v = 0 → v₂ = (1, −1).

These two eigenvectors are the only directions that A acts on by pure scaling. If A is diagonalizable, these eigenvectors form the columns of P in A = PDP⁻¹.

Sources: intro_linear_algebra_notes, systems_row_reduction_notes
Tier 3 — Algebraically Grounded 5 chunks retrieved Source: eigen_notes
A nonzero vector v ∈ V is an eigenvector of a linear operator T: V → V if T(v) = λv for some scalar λ ∈ F, called the eigenvalue. The equation can be rewritten as (T − λI)(v) = 0, so an eigenvalue exists precisely when T − λI is not injective — i.e., ker(T − λI) ≠ {0}.

The eigenspace E_λ = ker(T − λI) is a nonzero subspace of V. Eigenvalues are coordinate-free invariants: since a change of basis replaces A by P⁻¹AP, and det(P⁻¹AP − λI) = det(A − λI), the characteristic polynomial p_A(λ) = det(A − λI) is preserved under similarity.

Eigenvectors for distinct eigenvalues are linearly independent. If V admits a basis of eigenvectors, T is diagonalizable: [T]_ε = diag(λ₁, …, λₙ). Otherwise, the Jordan normal form provides the canonical decomposition via generalized eigenspaces.

Sources: eigen_notes

Get Started

Setup

Runs entirely locally. Requires Python 3.12 and Ollama.

# 1. Create and activate virtual environment
python3.12 -m venv venv && source venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Pull Mistral
ollama pull mistral:instruct

# 4. Build the knowledge base (one-time)
python build_kb.py

# 5. Launch the UI
python interface/app.py
